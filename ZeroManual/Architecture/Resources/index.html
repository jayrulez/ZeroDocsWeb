Resources

Resources are where most data comes from in a game. While Cogs allow us to model any scene we want, the model will often lack context or depth with out resources such as visual and audio assets. While many resources are imported into a project from outside the engine some such as Levels and Archetypes are created within the engine.

  • Read about the concept of resources here
  • How to add a resource to a project
  • Organize groups of resources at runtime with Resource Tables

Independent Resources

While most resources are directly tied to larger systems within the engine and are therefore included in the documentation for the respective system (e.g., the page for GeneratedSound can be found in the Audio index), there are a few that are independent of any one system. As a matter of convenience, these particular resources are collected here:

  • ColorGradient
  • Font
  • Level
  • Path
  • SampleCurve
  • TextBlock

Types of Resources

Resources vary greatly in type, ranging from audio and visual assets to physics materials and meshes to different types of Zilch files.

Visual Resources

  • SpriteSource
  • TextBlock
  • Font
    • Used exclusively with SpriteText to change the font to a user-imported one.
  • Texture
  • Material
  • ColorGradient
    • A user-defined color gradient from which samples may be taken

Audio Resources

  • Sound
  • SoundCue
  • SoundTag
  • GeneratedSound
    • Generates tones based on user-defined properties.

Physics Resources

  • PhysicsMaterial
  • PhysicsMesh
  • CollisionGroup
    • Used to separate objects with collision into discrete groups that may have custom collision properties applied between them
  • CollisionTable
    • The table used to hold all CollisionGroups and set collision properties between each group pairing
  • ConvexMesh
    • The mesh used to define the boundaries of ConvexMeshColliders
  • MultiConvexMesh
    • The mesh used to define the boundaries of MultiConvexMesh Colliders
