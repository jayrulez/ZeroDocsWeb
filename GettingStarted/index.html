Getting Started

Welcome to the website for the Zero Engine, DigiPen Institute of Technology‘s proprietary game engine and editor. The Zero Engine is a powerful simulation engine, custom built in C++ by DigiPen’s Research and Development Team (a group of DigiPen graduates and instructors).

Zero Engine Features

The Zero Engine can be used to make 2D and 3D games, as well as simulations of any kind. Projects range from simple prototypes to realistic simulations and professional games. Versatility of the engine is made possible by the following features:

  • A sophisticated, easy-to-use editor: ZED (Zero Editor)
  • Advanced component-based architecture
  • Support for 2D and 3D games
  • Custom-built constraint-based physics engine
  • Realistic flight modeling
  • Dynamic 3D lighting
  • Dynamic shader compositing

Learning the Zero Editor

We understand that not all users have the same level of experience. So we have tried to outline the most efficient path for you to become productive in Zero.

Beginner Users

If you have never made a game or done any programming before, you can still do quite a bit with the Zero Editor.

  • Start by going through our intro curriculum for the Engine. This will walk you through a majority of the basics of the Zero Engine in tutorial fashion.
  • Refresh yourself by reading the Editor sections.
  • If you start feeling like you are in over your head on the programming side, check out the Zilch in Zero Zero Manual section to really master Zilch.
  • Once you are comfortable with scripting in Zilch read the scripting, architecture, and gameplay sections of the Zero Manual.
  • While not always required, the physics, graphics, audio, and user interface sections are all considered core knowledge when developing in Zero.
  • If you feel like you have a grasp on the basics of how to implement gameplay functionality, try practicing a little with some simple feature tutorials.
  • When you feel somewhat comfortable with the code, go through the articles section to get a better sense of the common pitfalls and higher level paradigms in the engine.
  • Finally, start making your own games!
  • Remember that you can always look things up in the code reference, ask your peers for help, and get developer support.

Intermediate Users

If you have some experience making games and programming, you will be able to pick things up fairly quickly.

  • Start by checking out the Zilch in Zero Zero Manual section brush up on Zilch syntax.
  • Read the Zero Manual starting with the Editor, architecture, and gameplay sections to get oriented in the engine.
  • Browse physics, graphics, audio, and user interface sections for topics that are applicable to your project.
  • Read the remaining Zero Manual sections at your own pace, as they are often only used in more advanced projects.
  • If you are not fully confident in your ability to start operating in the engine try out some feature tutorials.
  • Finally, start making your own games!
  • Remember that you can always look things up in the code reference, ask your peers for help, and get developer support.

Advanced Users

If you are an experienced programmer/designer who has used component-based engines before, you will feel right at home.

  • Skim the Zilch in Zero Zero Manual section to brush up on Zilch syntax which should be familiar to those who have worked in ActionScript, C-Sharp, or any C-based language.
  • Read the Zero Manual starting with the Editor, architecture, and gameplay sections to get oriented in the engine.
  • Read the remaining Zero Manual sections at your own pace. While some such as physics, graphics, audio, and user interface will most likely be used in your projects, you will most likely have general knowledge about these topics from previous work.
  • If you are not fully confident in your ability to start operating in the engine try out some feature tutorials.
  • Finally, start making your own games!
  • Remember that you can always look things up in the code reference, ask your peers for help, and get developer support.
